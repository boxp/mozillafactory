イベント開催

概要

キッチンテーブル イベントの楽しみ方:

  • キッチンテーブルの周りに 友人と集まり ます
  • Web 上に何かを作ります

このガイドが助けになるでしょう。これは、コミュニティによるドキュメント です。ぜひ、あなたのアイデアを共有 し、他の人が開いたイベント を参考にしてください。

イベントの外観
  • 1 ～ 5 人の友人
  • 15 分～ 1 時間
  • 自宅やカフェなど

準備

何を作るかを決めよう
  • はじめに、Web 上に何かを作る プロジェクトを選択 しましょう。
  • ガイドラインを読んでイベント環境を整えましょう。
  • 参加者の 興味のある内容やスキルレベル を考慮し、適切なプロジェクトを選びましょう。
  • 何か違うことをしようとする場合は、それが どのように行われたか教えてください。
場所と日時を決めよう
  • 快適で、自由に使える場所 を見つけてください。ご自宅のキッチンやリビング、または近くのカフェなどでもよいでしょう。椅子を移動して円形に座れる形にしてください。
  • 無線 LAN などのインターネット回線と電源 を確保し、明るい場所でお菓子があるとよいでしょう。
  • 日時を決めてください。あなた自身が十分な時間を確保し、参加者が集まれる時間にしましょう。
友人を招待しよう
  • 短い招待状を書いてください。必要な持参するものがある場合は、そのことも忘れずに書いてください。
  • 下記の 例文 を使うとよいでしょう。
  • おしゃれな招待状を送りたいときは、Hackasaurus の X-Ray ゴーグル を使って、この招待状 をリミックスしてみましょう。

(ご友人) 様

Web サイトを作り変えませんか？ (日時) に (場所) でイベントを開きます。

私は Web ページのコードを学んで作れるようになりたいと思っています。あなたもご一緒に学びませんか？

私たちは、(日時)、オンライン上に何かを作るために (場所) に集まります。

おいしいお菓子とコードを学ぶ友人たちが集まる予定です。

できれば、ご自分のコンピュータを持参してください。

もし参加できるようであれば教えてください。

イベント中

挨拶をしよう
  • 全員がコンピュータと電源、インターネットを利用 できるようにしてください。
  • お互いのことをまだ知らなければ、自己紹介 をしましょう。
  • イベントの概要を簡潔に説明 してください。今日は何を作りますか？
プロジェクトを試そう
  • 以前に選択した Webmaker プロジェクト を実演しましょう。
  • 次は、ご友人に試してもらう 番です。
  • 全員に試してもらいましょう。次のステップに進む前に質疑応答をして、グループの準備を整えてください。
Web ページのコードを書き換えよう
  • 誰かが他のことをしたい場合は、別の Webmaker project を選ぶこともできます。
  • オンライン上の参考になるサイト も、参加者の興味とスキルレベルに合わせて利用してください。これらのサイトにも、すばらしいドキュメントとチュートリアルがあります:
    • Peer to Peer University (P2PU)
    • Mozilla Developer Network (MDN)
    • Code Academy
  • 参加者が正しく作業できているか 見て回りましょう。
成果を共有しよう
  • コードの書き換えが終わったら、"Publish" を押しましょう。
  • 出来上がったものを 友人たちに披露 して、お互いに見せ合いましょう。
  • 気に入ったら、書き換えたページを世界中の人たちと共有しましょう！
    • こちらの etherpad にURL を貼り付けてください。
    • または、#mozparty タグを付けて URL をツイート しましょう。
    • または、Flickr などの写真共有サイトで #mozparty タグを付けて 画像をアップロード しましょう。

イベント後

まとめ
  • 全員に今日の成果を尋ねましょう。イベントとプロジェクトについてのフィードバックを得るようにしてください。彼らが何を学んだか、何が良かったか？ イベントをより良くする提案があれば、ノートに書き留めてください。
  • このイベントを続けるか話し合ってください。あなた自身がこれらのプロジェクトでイベントを行うか、後で別のものを試してみたいかもしれません。別のキッチンテーブルイベントなど、ご友人が次回したいことを話し合ってください。
  • 興味があれば、次回の日時を設定 しましょう。開催日を選んでください！
イベントの経過を報告してください
  • イベント開催やプロジェクトについて、作ったものへのリンクを含めて イベントでの経験を共有 しましょう。
  • イベント開催後、フィードバックを集めるために 私たちからメールを送信 します。
